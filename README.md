 ## E-Commerce Customer & Revenue Analytics (RFM)
## Project Overview
This project is an end-to-end data analytics solution built to analyze an e-commerce business using PostgreSQL, Python, and Power BI.
It focuses on sales performance, customer behavior, and customer segmentation using the RFM (Recency, Frequency, Monetary) framework.

## Business Objectives
Analyze overall sales and revenue performance
Identify high-value and at-risk customers
Understand customer purchase behavior
Support data-driven marketing and retention strategies

## Tools & Technologies

PostgreSQL – Database design and SQL analysis
Python (Pandas, SQLAlchemy, Matplotlib, Seaborn) – Data cleaning, RFM analysis, segmentation
Power BI – Interactive dashboards and storytelling
GitHub – Version control and portfolio presentation

## Dataset
Multi-table e-commerce transactional dataset
Includes customers, orders, order items, payments, products, reviews, sellers, and geolocation data

## Project Architecture
PostgreSQL (Raw Tables & SQL Views)
        
Python (Data Validation & RFM Segmentation)
        
CSV Exports
        
Power BI Dashboards

## Key Analysis Performed

### Sales & Revenue Analysis (SQL)

* Monthly revenue trends
* Total revenue, orders, customers, and AOV
* Top product categories and cities by revenue

### Customer Segmentation (Python – RFM)

* Recency, Frequency, Monetary calculations
* Quintile-based RFM scoring
* Customer segmentation into:
  * Champions
  * Loyal Customers
  * Potential Loyalists
  * At Risk
  * Lost Customers

### Business Insights
* Revenue contribution by customer segment
* Purchase frequency distribution
* Identification of marketing target groups-
